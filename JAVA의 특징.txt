자바의 문법
	객체지향언어(Object-Oriented Programming)의 특징
		- 캡슐화
		- 다형성
		- 추상화
		- 상속

	접근제어자
		- public
			어디에서나 접근 가능
		- private
			클래스 내에서만 접근 가능
		- protected
			상속받은 클래스이거나 같은 패키지에서만 접근 가능
		- default
			동일 패키지에서만 접근 가능
	
	자바8 이후 함수형 프로그래밍 도입
		- 람다 표현식
		- 스트림API

자바의 도구
	- JSheel
		REPL(Read-Eval_Print Loop) 환경구축
		실시간 확인 가능
	- 자바의 빌드
		자바소스코드 => 자바 컴파일러 	=> 자바 바이트코드 => 자바실행환경(JRE)
		WORA(한번 쓰면 어디에서든 빌드가능)
		1. 자바 소스 파일(.java)을 자바 컴파일러(javac)가 컴파일하여 바이트 코드(.calss)를 만든다.
		2. 각 운영체제별 JRE(자바 실행 환경)에서 JVM(자바 가상 머신)을 통해 바이트 코드(.class)를 운영체제에 맞게 해석하여 실행한다.
